# 🚀 Talgat Grami - Personal Portfolio Website

Welcome to the repository for my personal portfolio website! I built this modern, responsive, single-page application to showcase my skills, professional experience, and featured projects as an **AI/ML Engineer**.

🌐 **[View Live Demo Here]** *(Add your hosted link here, e.g., GitHub Pages)*

## ✨ Key Features

- **Modern UI/UX:** Built with a clean, professional aesthetic tailored for a tech professional.
- **Dark & Light Mode:** Fully functional theme toggle that respects user system preferences and saves to local storage.
- **Dynamic Experience Counter:** A custom JavaScript function that automatically calculates and updates my years of professional experience down to the decimal.
- **Interactive Animations:** Features a custom CSS floating particle background and smooth hover scaling effects.
- **Working Contact Form (No Backend):** Integrated with EmailJS to send messages directly from the webpage to my inbox without needing a dedicated server.
- **Smooth Navigation:** Internal anchor links feature smooth scrolling for a seamless user experience.
- **Responsive Design:** Fully mobile-optimized layout using Tailwind CSS.

## 🛠️ Tech Stack

- **HTML5** for structure.
- **Tailwind CSS** (via CDN) for rapid, responsive styling.
- **Vanilla JavaScript** for logic (theme toggling, dynamic calculations, form handling, and particle generation).
- **EmailJS** for handling contact form submissions.
- **FontAwesome** for scalable vector icons.
- **Google Fonts** (`Inter` and `Space Grotesk`) for typography.

## 📂 Project Structure

The entire portfolio is designed to be lightweight and fast, contained primarily within a single file:
- `index.html`: Contains all the HTML structure, Tailwind utility classes, custom CSS animations, and JavaScript logic.
- `Talgat_Grami_Resume.pdf`: The local resume file triggered for direct download.

The contact form uses EmailJS. If you fork this project to use for yourself, you will need to create your own free EmailJS account and replace the PUBLIC_KEY, SERVICE_ID, and TEMPLATE_ID in the <script> section at the bottom of the HTML file with your own credentials.
